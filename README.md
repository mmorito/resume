## 職務経歴

### 株式会社エムネス: 2016.4 ~ 現在

#### 開発チームの新規立ち上げ / フルスタックエンジニア: 2016.4 ~ 2017.12

【当時の背景 / 自身の役割】  
医療支援プラットフォームの新規開発を行うため、初めてのエンジニアとして入社し2人で開発部門の立ち上げを行いました。  
このプラッフォームは、撮影されたCTやMRIなどの医用画像をクラウド上にアップロードし無制限に長期保管したり、web上で専門医に診断依頼を出すことができる医療機関や診断機関向けのクラウドサービスです。  

開発部門の創設メンバーとして自社プロダクト開発をローンチするため、文字通りフルスタックに様々な業務を担当しました。

当初、Webアプリケーションの開発経験が豊富とは言えなかったことや開発スピード・コストを鑑みて、外部パートナーとしてオフショアチームに参画頂き、  
社内の医師やコメディカルから要件をヒアリングしながら、技術選定・設計・開発、サービスのローンチ・運用/保守まで全工程を主導しました。  


【自身が発揮したバリュー及び成果】

**スタートアップでの新規プロダクトローンチ**  
2人で開発部門を立ち上げ、オフショアチームと連携し、当時ではかなりモダンな技術の採用にも挑戦しながら新規プロダクトの開発からローンチまでを完遂しました。  
現在は1000を超える施設で利用されるプロダクトに成長し、累計データ量は10億件を突破、約100万枚/日の画像データを処理しています。  
Goの経験はありませんでしたが、App Engine との親和性を鑑みてバックエンド開発言語として採用し、オートスケールの恩恵を最大限受けられるようにしました。

**大量の画像データの解析を行うためのアーキテクチャ設計**  
Pub/Sub と Kubernetes Engine を採用して画像解析を分散処理するための基盤を構築することで、1回の検査で撮影される1〜1000枚程度の画像が複数施設から断続的にアップロードされても、3分以内にすべての画像解析を行うことが可能になりました。  
また、NoSQL データベースである Cloud Datastore を採用することで、大量のメタデータを一貫性やパフォーマンスを保ちながら管理する仕組みを構築しました。


【その他】  
スタートアップならではの環境で多くのことをカバーする必要がありましたが、業務範囲を決めることなく、  
開発部門の枠組みを超えてビジネス面の検討や営業同行、展示会出展対応に至るまで必要だと感じた業務になんでも挑戦しました。

#### Senior Software Engineer/SRE (1): 2021.10 ~ 2023.9

【当時の背景 / 自身の役割】  
プロダクトが成熟期を迎え、ユーザー数の増加に伴い信頼性やデータ基盤の構築が求められるようになったため、SRE/Platformチームを新たに立ち上げました。  
このチームでは、運用効率化やインフラ改善、データ基盤の構築などを担当しました。  


【当時の抱えていた課題等】  
プロダクトコードが肥大化してゆるやかに運用効率や信頼性が低下している感覚があったため、問題箇所を洗い出し効率化や信頼性向上を目指す必要がありました。  
また、蓄積したデータをうまく可視化や分析のために活用することができていなかったため、データ基盤を整備する必要がありました。  


【自身が発揮したバリュー及び成果】

**インフラの移行や見直し**  
App Engine 1st gen が End of Support を迎えるため、2nd gen への移行が必要でした。  
旧SDKからの脱却や一部の機能を代替サービスに移行する必要がありましたが、段階的なトラフィック切り替えなどを活用しながら最大 3K/sec のリクエストを処理するキャッシュサーバーをダウンタイムなしで移行しました。  
また、処理が効率的ではなかった画像解析基盤を Kubernetes Engine から Cloud Run へ移行することで、スループットの向上や大幅なコスト削減を実現しました。  

**簡易的なデータ分析基盤の作成**  
Cloud Datastore を BigQuery へ定期的にエクスポートし、Looker Studio や コネクテッドシートを活用してデータ可視化や分析を行うための基盤を整備しました。

#### Senior Software Engineer/SRE (2): 2023.10 ~ 現在

【当時の背景 / 自身の役割】  
既存プロダクトの運用/保守を行う部門と、新規事業の立ち上げを行う部門に体制が変更となりました。  
SRE/Platformチームの取り組みは継続しつつ、新規事業のプロジェクトマネージャーとしてMVP開発を主導することになりました。  
企画段階から参画し、新規事業の構想を具体化し、開発メンバーと連携しながらプロダクトのローンチを行いました。  


【当時の抱えていた課題等】  
新規事業の立ち上げにおいては、既存プロダクトとの連携やデータ基盤の構築、開発メンバーのスキルアップなど、多岐にわたる課題がありました。  
また、既存/新規プロダクト間のデータ連携や、マルチプロダクトでもコアリソースのデータは一元管理される必要があるなど、プラットフォーム全体としての課題もありました。  


【自身が発揮したバリュー及び成果】

**API基盤の構築と新規プロダクトのローンチ**  
どこかのタイミングまでは分散してデータを管理せざるを得ないとしても、将来的には一元管理できるようなAPI基盤が必要であることは明らかでした。  
そのため、会社にとってコアリソースと定義されるデータを複数のプロダクトから扱うため、マイクロサービスアーキテクチャを採用したAPI基盤を構築しました。  
API Gateway として Envoy を採用し、Protocol Buffers で定義したコアリソースを操作するためのAPI、Google Healthcare API をラップしたリバースプロキシなどをCloud Run 上に構築しました。  

更にこの基盤を活用して、複数の医療AIソフトウェアを利用する新規プロダクトのMVP開発を主導しました。  
APIの開発からインフラの構築までほぼ1人で担当しながら、ビジネスモデルの検討や協業ベンダーとの折衝、パートナー規約の整備なども行いました。  
市場の反響を得るため展示会の出展へ間に合うようローンチを行い、かなりチャレンジングなスケジュール感のなかでブース展示やセミナー登壇までやり切ることができました。

**API基盤とデータ基盤の継続開発**  
現在、社内の複数のプロダクトを横断的したプラットフォームの構築が急務と考えて、継続的に取り組みを主導しています。  
データ基盤においても、ETL・DWHを構成するコードやSQL、単発で提供するツール類に至るまでコード管理を徹底し、データの一貫性や品質を担保するための取り組みを行っています。
